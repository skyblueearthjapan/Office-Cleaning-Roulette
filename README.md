# 掃除ルーレット 🧹 (Office Cleaning Roulette)

社内の掃除場所を、誰がどこを担当するかルーレットで決める Google Apps Script (GAS) Web アプリです。
**スプレッドシートバインド型**で、裏側のスプレッドシートを DB として使います。掃除場所・スタッフ名は
スプレッドシートを編集するだけで変更できます。

## 仕組み

- 外側のリング = **掃除場所**（固定）
- 内側の円盤 = **スタッフ**（回転）
- 「ルーレットを回す」で各掃除場所にスタッフをランダムに割り当て、結果を一覧表示します。
- 実行結果は `履歴` シートに自動保存されます。

## スプレッドシート（DB）の構成

| シート名 | 用途 | 入力する場所 |
|---|---|---|
| `掃除場所` | 掃除場所の一覧 | A 列（1 行目はヘッダ） |
| `スタッフ` | スタッフ名の一覧 | A 列（1 行目はヘッダ） |
| `履歴` | ルーレット実行結果 | 自動追記（編集不要） |

- 掃除場所・スタッフを増減したいときは、各シートの **A 列に行を追加・削除** するだけです。
- スタッフ数が掃除場所数より少ない場合は循環して割り当て、多い場合は先頭から使います。
- シートが無い状態で初めて起動すると、既定値が自動で投入されます（スプレッドシートのメニュー
  **🧹 掃除ルーレット → マスタシートを初期化** からも作成できます）。

## ファイル構成

```
appsscript.json   GAS マニフェスト（タイムゾーン / Web アプリ設定）
Code.gs           サーバ側。スプレッドシート読み書き・Web アプリ入口
Index.html        画面（ルーレット UI / フロント JS）
.clasp.json       clasp 連携用（対象スクリプト ID）
```

## デプロイ手順

### A. clasp を使う場合（推奨）

```bash
npm install -g @google/clasp
clasp login
clasp push          # このフォルダの内容をスクリプトへ反映
```

その後、Apps Script エディタで **デプロイ → 新しいデプロイ → ウェブアプリ** を選択して公開します。

### B. 手動で貼り付ける場合

1. 対象スプレッドシートで **拡張機能 → Apps Script** を開く。
2. `Code.gs` / `Index.html` の内容を同名ファイルに貼り付け（`Index.html` は「ファイル → HTML」で作成）。
3. `appsscript.json` の内容をマニフェストに反映（表示 → マニフェストファイルを表示）。
4. **デプロイ → 新しいデプロイ → ウェブアプリ** で公開。

## 元デザイン

UI は Claude Design プロジェクト「社内掃除場所決めルーレット」の `掃除ルーレット.dc.html` を基に、
GAS の HtmlService 向けに実装し直したものです（データ駆動で掃除場所・スタッフ数に追従）。
